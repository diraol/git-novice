Controle de versão com Git

Hipólita e Hipátia foram contratatos pela Olympus Missions (uma spinoff de serviços espaciais da Euphoric State University) para investigar se é possível enviar o próximo veículo interplanetário delas para Marte. Elas querem poder trabalhar nos planos simultaneamente, mas elas passaram por problemas com isto no passado. Se elas se revesarem, cada uma irá perder muito tempo esperando a outra finalizar, mas se elas trabalharem em suas próprias versões e forem enviando-as entre si, muitas coisas se perderão, serão sobrescritas ou duplicadas.

Uma colega sugeriu utilizar controle de versão para gerir o trabalho. Controle de versão é melhor que enviar arquivos por email pois:

  • Nada que é comitado ao controle de versão é perdido. Já que todas as versões antigas dos arquivos são salvas, sempre é possível retornar no tempo para ver exatamente quem escreveu o quê num dia específico, ou qual versão de um programa foi utilizada para gerar um determinado conjunto de resultados.

  • Como temos este registro de quem fez qual mudança e quando, nós sabemos a quem perguntar se houverem dúvidas futuramente e, se necessário, reverter uma versão anterior, assim como o 'desfazer' em um editor de texto.

  • Quando muitas pessoas colaboram num mesmo projeto, é possível deixar passar ou sobrescrever as mudanças de alguém de forma acidental: o sistema de controle de versão notifica automaticamente a usuária sempre que há algum tipo de conflito entre versões diferentes.

Não são apenas times que se beneficiam do controle de versão: Pesquisadoras podem se beneficiar imensamente. Mantendo um registro do que foi alterado, quando, e o motivo é extremamente útil para todas pesquisadoras se elas já precisaram voltar a projetos (e.g., um ano depois, quando a memória já esqueceu).

Controle de versão é o caderno de anotações do mundo digital: é o que profissionais usam para manter registro do que fizeram e para colaborar com outras pessoas. Todo grande projeto de desenvolvimento de software depende disso, e a maioria das programadoras utilizam isso para seus pequenos trabalhos também. E não é apenas para softwares: Livros, papers, pequenas bases de dados, e qualquer coisa que mude ao longo do tempo ou precise ser compartilhada por ser armazenada num sistema de controle de versão.
